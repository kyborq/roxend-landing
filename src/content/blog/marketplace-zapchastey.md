## Задача

Компания-дистрибьютор автозапчастей хотела выйти на онлайн-рынок. Главный вызов — помочь покупателям находить нужные запчасти среди миллионов наименований.

Требования:

- Подбор запчастей по VIN-коду автомобиля
- Интеграция с каталогами производителей
- Умный поиск с учётом опечаток и синонимов
- Сравнение цен от разных поставщиков

## Решение

Мы создали полноценную маркетплейс-платформу с интеллектуальным поиском:

### Умный каталог

- Автоматический импорт данных из каталогов TecDoc и производителей
- Распознавание VIN-кода для определения комплектации
- Поиск по оригинальным и кросс-номерам
- Фильтрация по производителям и ценам

### Поиск на Elasticsearch

- Морфологический анализ русского языка
- Исправление опечаток
- Подсказки при вводе
- Поиск по фото запчасти (beta)

### Для продавцов

- Личный кабинет с аналитикой
- Массовая загрузка прайс-листов
- Автоматическое обновление остатков
- Интеграция с 1С

## Результаты

За первый год работы платформы:

- **100,000+** товарных позиций в каталоге
- **15** подключенных поставщиков
- **92%** точность подбора по VIN
- **2.5 сек** среднее время поиска

> «Раньше консультанты тратили по 10-15 минут на подбор запчасти. Теперь клиенты сами находят нужное за минуту.» — Руководитель проекта

## Архитектура

Система состоит из нескольких микросервисов:

- **Catalog Service** — управление товарами и категориями
- **Search Service** — Elasticsearch с кастомными анализаторами
- **VIN Decoder** — расшифровка и валидация VIN-кодов
- **Integration Hub** — синхронизация с внешними системами

Развёрнуто в Kubernetes для автоматического масштабирования при пиковых нагрузках.
