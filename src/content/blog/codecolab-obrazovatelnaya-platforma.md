## Проблема

Каждый преподаватель программирования сталкивается с одними и теми же проблемами:

- **Установка IDE** — на каждом компьютере в классе нужно установить и настроить среду разработки
- **Разные версии** — у студентов на домашних устройствах другие версии Python, Node.js или вообще их нет
- **Проверка заданий** — преподаватель получает файлы по почте, в мессенджерах, на флешках
- **Списывание** — невозможно понять, студент сам написал код или скопировал
- **Потеря кода** — студенты забывают сохранять, теряют файлы, не делают бэкапы

Мы решили создать платформу, которая устранит все эти барьеры и позволит сфокусироваться на главном — обучении.

## Решение

**CodeColab** — это веб-платформа для обучения программированию, которая работает прямо в браузере. Никаких установок, никаких настроек — открыл сайт и начал писать код.

### Онлайн-редактор кода

Полноценный редактор с подсветкой синтаксиса на базе CodeMirror 6:

- **8 языков программирования** — Python, JavaScript, C++, C#, PHP, Lua, Ruby, Pascal
- **Автосохранение** — код сохраняется каждую секунду, невозможно потерять работу
- **Мгновенное выполнение** — результат появляется за доли секунды
- **Поддержка stdin** — интерактивный ввод данных для консольных программ

### Безопасное выполнение кода

Код выполняется в изолированных Docker-контейнерах на удалённом сервере:

- Каждый запуск — отдельный контейнер
- Ограничения по CPU, памяти и времени выполнения
- Полная изоляция от других пользователей
- Автоматическое удаление контейнера после выполнения

### Sharing проектов

Любой проект можно сделать публичным и поделиться ссылкой:

- Студент отправляет ссылку преподавателю
- Преподаватель видит код и результаты выполнения
- Режим просмотра — нельзя случайно изменить чужой код
- История всех выполнений сохраняется

### Аналитика для преподавателей

Главная фича для образовательных учреждений — детальная аналитика активности студентов:

- **Keypresses** — сколько клавиш нажал студент (throttle 5 сек)
- **Copy/Paste** — отслеживание копирования и вставки с метаданными
- **Тепловые карты** — когда студент работал над проектом (по дням и часам)
- **Сессии** — длительность работы, количество запусков, автосохранений
- **Подозрительная активность** — высокий ratio копирования, короткие сессии с большим кодом

Преподаватель может объективно оценить, сколько усилий студент вложил в задание.

## Архитектура

### Frontend

- **React 19** + **TypeScript** — современный UI с типизацией
- **CodeMirror 6** — редактор кода с подсветкой синтаксиса для всех 8 языков
- **TanStack Query** — кэширование и синхронизация данных
- **Recharts** — визуализация аналитики (графики, тепловые карты)
- **Emotion** — стилизация компонентов

### Backend

- **Appwrite Cloud** — Authentication, Database, Real-time subscriptions
- **Cookie-based sessions** — безопасная авторизация
- **NoSQL коллекции** — Projects, Executions, Sessions, Events
- **Ролевая модель** — обычные пользователи и преподаватели (label: teacher)

### Executor Service

- **Node.js + Express** — HTTP API для выполнения кода
- **Docker SDK** — программное управление контейнерами
- **JWT Authentication** — проверка токенов Appwrite
- **Resource limits** — ограничения CPU, памяти, времени

### Landing

- **Astro** — статический генератор для максимальной производительности
- **Roadmap с голосованием** — пользователи влияют на приоритеты разработки
- **SEO-оптимизация** — Open Graph, JSON-LD, мета-теги

## Внедрение

Платформа активно используется в реальном образовательном процессе.

### Марийский государственный университет (МарГУ)

CodeColab применяется при преподавании курсов по программированию:

- **Студенты университета** — основные курсы по программированию
- **Школьники 10-11 классов** — дополнительное образование, обучение Python

Преподаватели используют аналитику для отслеживания прогресса и выявления тех, кто нуждается в дополнительной помощи. Отсутствие необходимости устанавливать ПО особенно важно — студенты и школьники работают на любых компьютерах, включая домашние устройства.

### Преимущества для образовательных учреждений

- **Единая среда** — все студенты работают в одинаковых условиях
- **Нет технических барьеров** — не нужно устанавливать Python, IDE, библиотеки
- **Прозрачность работы** — преподаватель видит реальную активность каждого студента
- **Доступность 24/7** — студенты могут практиковаться из дома в любое время
